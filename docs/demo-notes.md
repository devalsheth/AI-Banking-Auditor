# Demo Notes

## Suggested flow
1. Start backend and frontend.
2. Show synthetic banking audit scenarios.
3. Regenerate data to prove dynamic behavior.
4. Open a high-risk item and explain drivers.
5. Ask the agent a domain-specific question.
6. Mention FAISS retrieval over audit and regulatory context.

## Domain focus
- Banking and financial institutions
- KYC/AML
- Treasury and liquidity
- Credit and NPA review
- Branch operations
- ITGC and privileged access
- Regulatory reporting
